## Recommendations

### Product Positioning:
1. **Primary Message:** "AI-powered workflow automation for technical GTM teams"
2. **Secondary Message:** "From prospecting to enrichment, automate your entire go-to-market stack"
3. **Differentiation:** Python-level customization + MCP integration for advanced users

### Marketing Channels:
1. **LinkedIn:** Primary channel (all profiles active)
2. **Twitter/X:** For AI/automation founder community
3. **Technical Communities:** Clay community, AI agent forums, GTM Slack groups
4. **Content:** Technical tutorials, use case breakdowns, workflow templates

### Sales Approach:
1. **Self-Service First:** Technical users want to explore and test independently
2. **Use Case Templates:** Provide pre-built workflows for common scenarios
3. **Integration Focus:** Highlight MCP, API access, Python SDK
4. **Community Building:** Create spaces for users to share workflows and learnings

### Product Development:
1. **Clay Integration:** 10% are Clay users (Miska is Clay Creator)
2. **MCP Tool Ecosystem:** Expand available tools and connectors
3. **Workflow Templates:** Pre-built solutions for prospecting, enrichment, routing
4. **Documentation:** Technical docs for advanced customization

## ICP Summary Statement

**Our ideal customer is a technically-savvy founder, GTM professional, or engineer (25-45 years old) with 5-15 years of experience, likely holding an advanced degree in a technical field. They're building or scaling AI-powered automation solutions, have previous experience at enterprise tech companies or are running their own startups/agencies, and are active in the AI/automation community. They're looking for a flexible, Python-level customizable platform to automate prospecting, enrichment, and GTM workflows that can integrate deeply with their existing tech stack via MCP and APIs.**
